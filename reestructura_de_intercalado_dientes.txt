# Arreglar intercalado correcto de dientes entre dos engranes externos

Tenemos un problema con el intercalado visual de los dientes. Actualmente se está intentando resolver con una lógica de “snap” al diente más cercano usando `Math.round(meshAngle / pitchAngle)`, pero eso es incorrecto o frágil.

Quiero reemplazar esa lógica por un sistema determinista de fase inicial.

## Problema actual

La lógica actual hace algo como:

```ts
const k = Math.round(meshAngle / pa1);
const driverToothAngle = k * pa1;
const contactDir = driverToothAngle + Math.PI;

const j = Math.round((contactDir - pa2 / 2) / pa2);
const valleyAngle = j * pa2 + pa2 / 2;

g2Phase = contactDir - valleyAngle;
````

Esto falla porque:

1. Está ajustando el ángulo geométrico al diente más cercano.
2. Introduce discontinuidades cuando cambia el número de dientes.
3. Puede producir saltos visuales.
4. No garantiza una convención clara de fase.
5. No debe ser necesario buscar el diente más cercano: la rotación inicial debe definir directamente qué diente apunta al contacto.

## Nueva regla correcta

Para dos engranes externos, si el engrane A está en `(xA, yA)` y el engrane B está en `(xB, yB)`, entonces:

```ts
const meshAngle = Math.atan2(yB - yA, xB - xA);
```

Este ángulo apunta desde A hacia B.

El ángulo opuesto, desde B hacia A, es:

```ts
const oppositeMeshAngle = meshAngle + Math.PI;
```

Quiero que en la posición inicial:

* El engrane A tenga un diente apuntando hacia B.
* El engrane B tenga un valle apuntando hacia A.

Esto debe cumplirse exactamente, sin `Math.round`.

# Convención geométrica obligatoria

Primero define claramente esta convención en el código:

Cuando `rotationRad = 0`, el engrane tiene el centro de un diente apuntando hacia `0 rad`, es decir, hacia `+X`.

Entonces:

```ts
toothCenterLocalAngle(k, teeth) = k * 2π / teeth
valleyCenterLocalAngle(k, teeth) = (k + 0.5) * 2π / teeth
```

Es decir:

* dientes en: `0`, `1*pitch`, `2*pitch`, etc.
* valles en: `0.5*pitch`, `1.5*pitch`, `2.5*pitch`, etc.

Si el generador actual del path no cumple esto, corregirlo. El diente 0 debe quedar centrado en `0 rad`.

# Fase inicial correcta

Crear una función:

```ts
type MeshPhaseParams = {
  driverTeeth: number;
  drivenTeeth: number;
  meshAngleRad: number;
};

type MeshPhaseResult = {
  driverInitialRotationRad: number;
  drivenInitialRotationRad: number;
};
```

Implementar:

```ts
export function calculateExternalGearInitialPhase({
  driverTeeth,
  drivenTeeth,
  meshAngleRad,
}: MeshPhaseParams): MeshPhaseResult {
  const drivenPitch = (2 * Math.PI) / drivenTeeth;

  // Queremos que un diente del driver apunte exactamente hacia el driven.
  // Como el diente 0 apunta a +X cuando rotation = 0:
  const driverInitialRotationRad = meshAngleRad;

  // Queremos que un valle del driven apunte exactamente hacia el driver.
  // El ángulo desde driven hacia driver es meshAngle + PI.
  // El primer valle local del driven está en drivenPitch / 2.
  const drivenInitialRotationRad =
    meshAngleRad + Math.PI - drivenPitch / 2;

  return {
    driverInitialRotationRad: normalizeAngle(driverInitialRotationRad),
    drivenInitialRotationRad: normalizeAngle(drivenInitialRotationRad),
  };
}
```

También implementar:

```ts
export function normalizeAngle(angle: number): number {
  const twoPi = 2 * Math.PI;
  return ((angle % twoPi) + twoPi) % twoPi;
}
```

No usar `Math.round` para calcular la fase de intercalado.

# Animación correcta

Una vez calculadas las fases iniciales, la animación debe conservar la relación cinemática.

Para engranes externos:

```ts
drivenRotation = drivenInitialRotation - driverDelta * driverTeeth / drivenTeeth;
driverRotation = driverInitialRotation + driverDelta;
```

Implementar algo como:

```ts
export function getExternalGearAnimatedRotations(params: {
  driverInitialRotationRad: number;
  drivenInitialRotationRad: number;
  driverDeltaRad: number;
  driverTeeth: number;
  drivenTeeth: number;
}) {
  const {
    driverInitialRotationRad,
    drivenInitialRotationRad,
    driverDeltaRad,
    driverTeeth,
    drivenTeeth,
  } = params;

  return {
    driverRotationRad: driverInitialRotationRad + driverDeltaRad,
    drivenRotationRad:
      drivenInitialRotationRad -
      driverDeltaRad * (driverTeeth / drivenTeeth),
  };
}
```

Importante:

* No recalcular la fase inicial en cada frame.
* Calcular la fase inicial solo cuando cambien:

  * número de dientes
  * módulo
  * posición relativa de centros
  * tipo de conexión
* Durante la animación solo cambia `driverDeltaRad`.

# Distancia entre centros

La distancia entre centros debe salir de los radios pitch, no de una posición visual arbitraria.

Para dos engranes externos:

```ts
pitchRadiusA = moduleMm * teethA / 2;
pitchRadiusB = moduleMm * teethB / 2;

centerDistance = pitchRadiusA + pitchRadiusB;
```

Equivalente:

```ts
centerDistance = moduleMm * (teethA + teethB) / 2;
```

Si el usuario edita manualmente la distancia entre centros, entonces mostrar advertencia si no coincide con la distancia teórica.

La validación puede ser:

```ts
const expectedCenterDistance = moduleMm * (teethA + teethB) / 2;
const error = Math.abs(centerDistance - expectedCenterDistance);

const isValidMesh = error < 0.001;
```

Si `isValidMesh` es falso, los engranes pueden tocar mal visualmente aunque la fase esté bien.

# Importante sobre SVG

Si el sistema usa coordenadas SVG donde `y` crece hacia abajo, tener cuidado con el signo de la rotación.

Revisar si una rotación positiva en el render se ve horaria o antihoraria.

Pero no mezclar esta corrección dentro de la matemática base.

La geometría debe trabajar en coordenadas matemáticas normales:

* `+X` derecha
* `+Y` arriba
* rotación positiva antihoraria

Luego, al convertir a SVG, aplicar la transformación correspondiente.

Si actualmente se genera directamente en coordenadas SVG con `y = cy + sin(angle) * r`, entonces documentar que la rotación visual está invertida y ajustar el signo de animación si hace falta.

# Revisar el generador del path

Verificar que el engrane generado realmente tenga un diente centrado en `0 rad`.

Si el path actual empieza con un diente en `-π/2`, como en versiones anteriores, entonces la fase anterior no va a funcionar.

En ese caso hay dos opciones:

## Opción A, recomendada

Cambiar el generador para que el primer diente esté centrado en `0 rad`.

Es decir, eliminar offsets como:

```ts
center = -Math.PI / 2 + i * pitchAngle;
```

y usar:

```ts
center = i * pitchAngle;
```

## Opción B

Si se quiere conservar el primer diente apuntando hacia arriba, entonces definir:

```ts
const baseToothOffset = -Math.PI / 2;
```

y compensar todas las fases:

```ts
driverInitialRotationRad = meshAngleRad - baseToothOffset;
drivenInitialRotationRad =
  meshAngleRad + Math.PI - drivenPitch / 2 - baseToothOffset;
```

Pero prefiero la opción A porque simplifica todo.
